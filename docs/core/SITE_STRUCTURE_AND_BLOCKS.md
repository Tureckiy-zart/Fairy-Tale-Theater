## 1. Принципы структуры

1. **Конверсия — первична.** Главная цель сайта — заявка/звонок. Лид-захват присутствует на каждой странице (CTA «Book» в шапке + блок-бэнд внизу + click-to-call). Заменяем ручной обзвон постоянным каналом.
2. **Один сайт, 4 линии услуг — секции под зонтиком Miss Lana** (`BRAND.md`): Fairy-Tale Theater (флагман) · Birthday Parties · School Shows · & Friends. Не пять сайтов.
3. **Широко в услугах, узко в идентичности.** Везде ведём «профессиональный живой костюмированный театр сказок», услуги (дни рождения/персонажи) — форматы под этим зонтиком.
4. **Две аудитории на одной странице.** Директор сада/школы (B2B: профессионализм, ценности/SEL, «под ключ», бюджет) и родитель (B2C: тёплый волшебный праздник). Дуальная подача — рядом «довод-доверие» и «довод-волшебство» (модель Lovevery).
5. **Английский-первый.** Опциональный украинский слой — позже, по решению владельца (`hreflang`).
6. **SEO-структура.** Каждое шоу и каждая услуга — **отдельный индексируемый URL** (не один SPA-экран); бренд сверху объединяет.
7. **Реальные фото ведут.** Иллюстрация/мотив света — обрамляет, не перекрывает лицо и не вытесняет фото (`DESIGN_SYSTEM.md` §9).
8. **Доступность и скорость встроены.** WCAG AA + Core Web Vitals (LCP — hero-фото без анимации; motion-safe-first).
9. **Честность.** Не «единственные/первые/конкурентов нет». Дифференциатор — живой театр + ценности + 30+ лет + труппа.
10. **Mobile-first.** Базовые стили — под мобильный экран; вверх масштабируем брейкпоинтами Tailwind (`md:`/`lg:`). Все страницы и блоки строятся mobile-first по умолчанию (`DESIGN_SYSTEM.md` §2).

## 4. Постраничные блюпринты (стек блоков сверху вниз)

### 4.1 Home (`/`) — самая важная

1. **Hero** `[ASSET]` — крупное тёплое фото/видео реального шоу (LCP, **без анимации**). H1 (Fraunces): «Live theater that comes to you». 2. **TrustStrip** `[READY]` — 5 быстрых маркеров доверия (для обеих аудиторий сразу).
3. **FormatExplainer** `[ASSET]` — что это: костюмированный спектакль ~30 мин + интерактив (пузыри), 35–50 мин, 2–10 лет; «настоящий театр, не один аниматор». Отстройка.

## 5. Конверсионная архитектура

- **Лид-захват везде:** «Book» в sticky-шапке + `BookingCTABand` внизу каждой страницы + click-to-call в шапке/футере/контактах.
- **Два пути:** B2B (сады/школы → `/school-shows`, акцент: профессионализм/ценности/«под ключ»/пакеты/уместность) и B2C (семьи → `/birthdays`, акцент: волшебство/тепло/без хлопот). Каждое шоу и линия ведут в одну общую `LeadForm`.
- **GBP — параллельный канал** (часто главный локальный): завести до запуска сайта; на сайте — блок зон/карта вторично.
- **Прозрачные цены** («от $350» + диапазон) снимают трение у B2B; скрывать = терять лиды.

## 6. SEO-структура по страницам (ref `04_SEO.md`)

- Отдельный индексируемый URL на **каждое шоу и услугу**; чистые англ. слаги; уникальные `title`/`description`/OG.
- Schema: `PerformingGroup`/`LocalBusiness` (`areaServed` = LA + Сан-Диего/Сакраменто/Сан-Хосе), `Event`/`TheaterEvent` на шоу, `BreadcrumbList`.
- Английский — по умолчанию; опц. украинский слой → корректный `hreflang` (en/uk).
- `/design` — **noindex**. Sitemap.xml/robots.txt; быстрый CWV (локальный ранг).
- Семантика — англ. интент (садики/ассамблеи/дни рождения/костюмированные шоу/bubble); подтвердить инструментом или Search Console после запуска.

**Позже:** `/blog`, `ReviewsBlock`, опц. украинский слой, гос-школьная GTM, листинги-агрегаторы.

**MVP-минимум для запуска:** каркас + Home + Booking + Pricing + Shows(hub+detail) + School-shows + Birthdays + Gallery + About. (`/services`, `/characters`, `/blog` — fast-follow.)
