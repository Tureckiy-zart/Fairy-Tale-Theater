# Client Intake Questionnaire — анкета первичного сбора

> Инструмент Фазы −1. Используется в начале работы с клиентом, чтобы собрать
> ПЕРВИЧНУЮ базу до ресёрча. Два режима — выбирается по ситуации:
>
> - **OUTSIDE-IN** (холодный заход на чужой бизнес): вопросы в основном К СЕБЕ —
>   понятная база, на которую можно ответить, глядя на бизнес снаружи. НЕ включает
>   то, что предстоит ресёрчить (конкурентов, SEO, рынок — это работа аналитика, не
>   вопрос клиенту).
> - **INSIDE-OUT** (есть контакт/сотрудничество с владельцем): вопросы К НОСИТЕЛЮ
>   бизнеса — собирается то, что знает только он (боль, цифры, голос бренда).
>
> Правило: НЕ спрашивай клиента о том, что можно выяснить самому. Анкета собирает
> то, что снаружи не видно. Всё видимое (конкуренты, рейтинги, сайт) — ресёрчится.

## Как пользоваться (оба режима)

1. **Outside-in**: режим A заполняется самостоятельно → ресёрч → отчёт → затем контакт с клиентом.
2. **Inside-out**: режим B отправляется клиенту → сбор ответов → ресёрч дополняет → отчёт.
3. **Гибрид** (частый случай): A заполняется сразу, B досылается, когда клиент готов сотрудничать.
4. **`CLIENT_QUESTIONS.md`** (в корне проекта, на языке клиента) — то, что скилл
   `bootstrap-project` собрал на Шаге 0: вопросы, на которые знает ответ только
   владелец (включая блок B7 «видение сайта»). Это готовый клиентский пакет режима
   B — отправляется владельцу; ответы возвращаются сюда, в discovery.
5. **Нет онлайн-следа** (ни сайта, ни Maps, ни соцсетей)? Минимального якоря
   **город + тип бизнеса** хватает, чтобы стартовать (карта конкурентов по типу+гео).
   Отсутствие каждого канала — находка (в отчёт), а недостающее уточняется у клиента
   через `CLIENT_QUESTIONS.md`. Метод не падает без якоря — деградирует на «собери
   что есть + остальное через клиента».
